Model Description:

What is it called: Subject Code

What its purpose is: Primary key to the database it identifies the object.

Data type: String

Any validation or constraint requirements: Consists of 3 letter and 3 numbers. The subject code must be places in the subjectCatalogue list



What is it called: Subject Name

What its purpose is: Identifies what the subject is called

Data type: String

Any validation or constraint requirements: Consists of only ASCII letters



What is it called: Semester

What its purpose is: Indicates What semester the subject is being tought

Data type: Integer

Any validation or constraint requirements: must be an integer value. Summer Semester is indicated as Semester 3.



What is it called: Discipline

What its purpose is: It describe's the department where the subject is tought.(Engineering or IT) It acts as a foreign key.

Data type: String

Any validation or constraint requirements: Must have a corasponding value in the second table to work as a foreign key.



What is it called: Description

What its purpose is: Describes what will be tought in the subject.

Data type: String

Any validation or constraint requirements: none
